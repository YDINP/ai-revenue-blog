## 개발자에게 마우스가 중요한 이유

개발자는 하루 8시간 이상 마우스를 사용합니다. 코드 편집, 브라우저 탭 전환, 디자인 리뷰, 문서 작업까지 — 마우스는 키보드만큼 중요한 도구입니다.

잘못된 마우스 선택은 **손목 통증(손목터널증후군)**으로 이어질 수 있습니다. 반대로 좋은 마우스는 생산성을 눈에 띄게 향상시킵니다.

이 글에서는 실제 개발 환경에서 테스트한 **개발자용 마우스 TOP 5**를 소개합니다.

## 1위: 로지텍 MX Master 3S — 개발자의 표준

**MX Master 3S**는 개발자 커뮤니티에서 가장 많이 추천되는 마우스입니다. 그 이유는 명확합니다.

### 핵심 장점

- **MagSpeed 스크롤**: 초당 1,000줄 스크롤 가능 — 긴 로그 파일이나 코드 탐색에 필수
- **3대 디바이스 전환**: 버튼 하나로 맥/윈도우/리눅스 즉시 전환
- **조용한 클릭**: 사무실이나 카페에서 민폐 없이 사용
- **USB-C 충전**: 1분 충전으로 3시간 사용 (급할 때 유용)
- **Logi Options+**: 앱별 버튼 커스터마이징 (VS Code에서 사이드 버튼 = 뒤로가기)

### 단점
- 가격이 높은 편 (10만원대)
- 손이 작은 분에게는 다소 큼

### 개발자 추천 설정

- **사이드 버튼 위**: 뒤로가기 (Alt+Left)
- **사이드 버튼 아래**: VS Code 명령 팔레트 (Ctrl+Shift+P)
- **제스처 버튼**: 데스크톱 전환
- **수평 스크롤**: IDE 탭 전환

## 2위: 로지텍 MX Ergo S — 손목 건강 최우선

이미 손목 통증이 있거나 예방하고 싶다면 **트랙볼 마우스**가 정답입니다.

### 핵심 장점
- **20도 각도 조절**: 손목의 자연스러운 자세 유지
- **트랙볼**: 마우스를 움직일 필요 없음 → 손목 부담 제로
- **120일 배터리**: 충전 걱정 거의 없음

### 단점
- 적응 기간 1~2주 필요
- 정밀 작업(디자인)에는 불편

## 3위: 로지텍 MX Anywhere 3S — 휴대성의 왕

외근이 많거나 카페 코딩을 즐긴다면 **MX Anywhere 3S**가 최적입니다.

### 핵심 장점
- **99g 초경량**: 어디든 가볍게 휴대
- **어떤 표면에서든 작동**: 유리 위에서도 정밀 추적
- **MagSpeed 스크롤**: Master 3S와 동일한 스크롤 성능

### 단점
- 장시간 사용 시 손목 지지력 부족
- Master 3S 대비 버튼 적음

## 4위: 라이젠 Pro Click Mini — 가성비 인체공학

소형 인체공학 마우스를 원한다면 좋은 선택입니다. 크기 대비 그립감이 우수합니다.

## 5위: 애플 Magic Mouse 3 — 맥 생태계 전용

맥 전용이라면 제스처 기능이 유용하지만, 충전 포트 위치(바닥)와 인체공학의 부재로 개발자에게는 비추천입니다.

## 손목 건강을 위한 추가 팁

1. **20-20-20 규칙**: 20분마다 20초간 20피트(6m) 먼 곳을 바라보기
2. **손목 스트레칭**: 2시간마다 1분 스트레칭
3. **마우스 패드**: 손목 받침대 있는 패드 사용
4. **마우스 위치**: 팔꿈치 높이에 마우스 배치
5. **키보드 단축키 활용**: 마우스 사용 최소화
